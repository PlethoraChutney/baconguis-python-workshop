Day Two: The terminal, math, and variables

The terminal

The terminal is a parallel way of doing things you might use a GUI for. Both are ways of talking to the Operating System.

You will often see scripts use the command line because you don't have to design an entire GUI for it, which would typically take much longer than writing whatever functionality is necessary otherwise.

The filesystem

Your filesystem is arranged like a "tree", with each directory representing a branch.

The tree starts, appropriately, at the "root" directory. In Mac/Linux this is simply /, while in Windows it's whatever the drive name is (typically C:\)

The filesystem

To describe where a file is, you can use an absolute path or a relative path.

Absolute paths describe the position with no ambiguity. They start from root and work their way all the way to the file.

Relative paths describe the position of the file relative to your current location. They start without the root.

Special paths

Both Windows and Mac/Linux use . to mean "here" and .. to mean "one directory above here"

Types

Types determine what types of operations are available for a given object, and how they are stored in memory.

Types: String

Everything we've been using print() on has been a string, or as it's known in python, a str.

Strings are collections of characters surrounded by quotes. Any time you want to store or use a word or sentence, that's a string 

python

Types: Integer

An integer, or int, is used to store whole numbers. This is what's used to do math, keep track of iterations, etc.

Types: Floats

To store large numbers in a memory-efficient way, or to store non-whole numbers, we need to use a float.

Python will keep track of when something needs to be converted to a float, there's no need to think about it yourself.

            >>> type(my_big_num)

            >>> print(my_big_num)

Types: Floats

It's impossible to represent all numbers with infinite precision in a finite system.

This is a well-known and well-studied problem in computer science. If you're interested, googling "floating point arithmetic" will get you more info on the topic.

For our purposes, it's sufficient just to know that this inaccuracy exists.

Types: Boolean

A boolean, or bool, stores a single binary value: True or False.

Even though these look like strings, they're special words in python to represent truth value, and so they don't need quotes.

Lists - slicing

You can access a sub-list by slicing a list

Slicing is an extension of the indexing syntax.

list[first:last:step]

First is inclusive, last is not. Step tells you the size of the step taken between first and last. 


my_list = [1, 2, 3, 4, 5]
print(my_list[0:3]) # [1, 2, 3]
print(my_list[0:5:2]) # [1, 3, 5]
# 4 is not printed below because
# the step size is too big
print(my_list[0:4:2]) # [1, 3]
# leaving out one of the options
# assumes sensible defaults
print(my_list[::2]) # [1, 3, 5]

Dictionaries

Syntax for accessing dictionary values looks very similar to list indexing, except you use the key instead of the index.

This can be confusing if you use ints as your keys; I'd recommend you avoid it if possible. 


            my_dictionary = {
                'key': 'value',
                'my_list': [1, 2, 3],
                1: 'one'
            }
            print(my_dictionary['key']) # 'value'

            # kinda confusing...looks like a list.
            print(my_dictionary[1]) # 'one'

Modifying collections

Up til now, we've been directly making our lists and dicts as we'd like to use them. But typically you'll want to modify collections by adding or removing elements or key/value pairs.

Modifying collections - Dicts

Adding an element to a dict is easy. Just assign it like any other variable. 

                my_dictionary = {
                    'key': 'value',
                    'my_list': [1, 2, 3],
                    1: 'one'
                }
                my_dictionary['hello'] = 'world!'
                print(my_dictionary)
                # result: 
                {
                    'key': 'value',
                    'my_list': [1, 2, 3],
                    1:'one',
                    'hello': 'world!'
                }

Modifying collections - Dicts

To delete an element, use the special del word. 

                my_dictionary = {
                    'key': 'value',
                    'my_list': [1, 2, 3],
                    1: 'one'
                }
                my_dictionary['hello'] = 'world!'
                del my_dictionary['my_list']
                print(my_dictionary)
                # result: 
                {
                    'key': 'value',
                    1: 'one'
                    'hello': 'world!'
                }

Modifying collections - Lists

To add an element to a list, we'll use our first method

Methods are like functions, except they're "attatched" to objects.

Methods are called like this: object.method(arguments) 


                my_list = [1, 2, 3, 4]
                # print() is a function.
                # It can be called directly
                print(my_list) # [1, 2, 3, 4]

                # append() is a method. It needs
                # to be attached to a specific list.
                #
                # Note that you don't need to
                # re-assign my_list.
                # It is modified in place.
                my_list.append(5)
                print(my_list) # [1, 2, 3, 4, 5]

Modifying collections - Lists

You can use del to remove a list element with its index. 


                my_list = [1, 2, 3, 4, 5]
                print(my_list) # [1, 2, 3, 4, 5]

                # remember that lists are
                # zero-indexted!
                del my_list[3]
                print(my_list) # [1, 2, 3, 5]

Conditionals

You can use any type in an if/else statement, since all types have a boolean evaluation. For instance, 0 is False while 12 would be True.

                number_of_eggs = 2

Scope

Variables defined inside a function are said to belong to the local scope of that function. They are not accessible outside that function.

Variables defined in the main body of a script are global, and can be accessed from any scope.

If you create a local variable with the same name as a global variable, python will treat them as separate objects. The local one will only be available in its own scope.
